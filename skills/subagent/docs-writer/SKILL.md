# Docs writer

Use this skill when the task itself is documentation work.

This skill defines documentation standards, editing workflow, and verification
steps. It is not an execution-agent skill.

## Scope

Use this skill when work involves:

- files in `/docs`,
- repository Markdown files,
- README updates,
- guides, tutorials, and explanations,
- documentation review and polishing.

## Writing principles

- Write for the reader using direct, clear language.
- Prefer active voice and present tense.
- Use concrete wording instead of vague marketing language.
- Distinguish requirements from recommendations clearly.
- Keep terminology consistent across edited files.

## Structure

- Start with a short introduction that explains what the page covers.
- Use clear hierarchical headings.
- Use numbered steps for procedures.
- Use bullets for non-sequential information.
- Add notes and warnings where they reduce user error.
- Add a next-steps section when it helps the reader continue.

## Preparation

Before editing:

1. Clarify the request if the goal is ambiguous.
2. Read the existing documentation.
3. Inspect relevant code or behavior for technical accuracy.
4. Check nearby or linked docs for consistency.
5. Form a short plan before making edits.

## Verification

Before finishing:

1. Re-read the edited content for accuracy and flow.
2. Verify links and references where relevant.
3. Ensure formatting matches repository conventions.
4. Run the formatter if the repository uses one and the user wants it run.
