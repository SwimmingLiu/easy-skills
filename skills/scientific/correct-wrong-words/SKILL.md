# Correct Wrong Words — 多格式错别字检测与修正

## 工作流程

按照以下 5 个阶段严格执行，不得跳过任何阶段：

### 阶段 1：格式解析与语法隔离

根据输入文本的格式类型，隔离所有非自然语言元素：

**Markdown 格式**（识别特征：`` ``` ``、`**`、`[text](url)` 等）：
- 跳过：代码块（`` ```...``` `` 或 `` `inline code` ``）、超链接 URL 部分（`[文字](URL)` 中仅处理"文字"）、图片路径、HTML 标签
- 保留处理：正文段落、标题文字、列表内容（纯文字部分）

**LaTeX 格式**（识别特征：`\`命令、`$`公式、`\begin{}`等）：
- 跳过：所有数学公式（`$...$`、`$$...$$`、`\begin{equation}...\end{equation}`、`\[...\]`）、宏命令（`\textbf{}`、`\cite{}`等的命令部分，保留大括号内的自然语言文字）、`\begin{}`/`\end{}` 环境声明、注释（`%` 后内容）、宏包引用
- 保留处理：`\section{这里的文字}`、`\caption{这里的文字}`、正文段落等大括号内的自然语言

**纯文本**：直接进入阶段 2。

> 🔑 **关键原则**：隔离的元素用占位符标记（如 `__SKIP_0__`、`__SKIP_1__`），处理完成后原样还原，不得修改任何被跳过的内容。

---

### 阶段 2：逻辑分块

将经过隔离处理后的自然语言文本，按**自然段落**（空行分隔）拆分为独立处理单元。每段独立进行阶段 3-4 的处理，避免超长文本导致注意力下降。

---

### 阶段 3：专业术语识别与保护标记

逐段扫描，识别以下类型的专业术语并用 `【术语: 原文】` 格式标记：

**需要标记的术语类型**：
- 医学/科学专有名词（如：肺结节、U-Net、YOLO、Transformer、mAP、IoU）
- 技术缩写与算法名（如：CNN、ReLU、BERT、ResNet、FPN）
- 特定软件/工具/框架名（如：PyTorch、TensorFlow、OpenCV）
- 专业领域术语（如：感受野、注意力机制、特征金字塔、语义分割）
- 人名、地名、机构名、期刊名

> ⚠️ **绝对禁止**：对被标记为术语的词汇进行任何修改、替换或润色。仅标记，不修改。

---

### 阶段 4：双语错别字识别与修正

对非术语的普通文字进行错别字扫描：

**中文错别字类型**：
- 同音字误用（如：的/地/得、在/再、做/作、已/以、己/已）
- 形近字混淆（如：己/已/巳、戊/戌/戍、宫/官、橫/衡、墒/熵）
- 词语搭配错误（如："即使…也"误用为"及使…也"）
- 常见混淆词（如：陈/沉、像/象、带/戴）
- **学术论文高频错误**：影象→影像、图象→图像、付合→符合、予以→予以、付出→付出、大副→大幅、平橫→平衡、交叉墒→交叉熵、严俊→严峻

**英文错别字类型**：
- 单词拼写错误（如：recieve→receive, occured→occurred）
- 单复数误用（如：informations→information）
- 动词时态拼写错误（如：writed→wrote）
- 常见混淆词（如：their/there/they're 明显错误使用）

> ✅ **操作规则**：直接替换修正，不做额外说明。仅修正明确的错别字，不做风格润色、词汇替换或语义修改。

---

### 阶段 5：无损还原与结构化输出

**5.1 无损还原**：
1. 将修正后的文本嵌回原始文档结构
2. 用对应的原始内容替换所有占位符（`__SKIP_0__` 等）
3. 严格保持原始的换行、缩进、标点格式

**5.2 结构化输出格式**（每个处理段落输出以下内容）：

```
--- [第 N 段] ---

[修正后的完整段落文本，术语已用【术语: xxx】标记]

📝 错别字修改记录：
• 原文"xxx" → 修改为"yyy"
• 原文"aaa" → 修改为"bbb"
（如无错别字，显示：✅ 本段无错别字）

🏷️ 识别到的专业术语：
• xxx、yyy、zzz
（如无专业术语，显示：无）
```

**5.3 汇总输出**：
在所有段落处理完毕后，输出一个总结区块：

```
=== 全文修正汇总 ===
📊 处理统计：共 N 段，发现并修正 X 处错别字，标记 Y 个专业术语

📝 全部错别字修改对照：
• 第N段：原文"xxx" → "yyy"
...

🏷️ 全文专业术语列表：
xxx、yyy、zzz...

📄 完整修正后文本：
[输出完整的、已修正的原格式文档]
```

**5.4 自动保存到项目临时目录**：

汇总输出完毕后，将结果保存为 Markdown 文件，便于后续查阅：

- **保存目录**：项目根目录下的 `.tmp/docs/temp/`（若目录不存在则先创建）
- **文件命名规则**：`typo-correction-{YYYY-MM-DD-HH-MM}.md`（使用当前时间戳）
- **保存内容**：完整的汇总输出（5.3 全文修正汇总区块的全部内容）
- **保存后提示**：在回复末尾告知用户文件完整路径，例如：
  > 💾 已保存至 `.tmp/docs/temp/typo-correction-2026-03-07-14-30.md`

> 🔑 **保存规则**：
> 1. 使用 Write 工具直接写入文件，不询问用户是否保存（静默执行）
> 2. 文件头部附加元数据注释，格式如下：
>    ```
>    <!-- 错别字修正记录 | 生成时间：YYYY-MM-DD HH:MM | 处理段落数：N | 修正错别字数：X -->
>    ```
> 3. 若用户明确说"不要保存"或"无需保存"，则跳过此步骤

---

## 执行注意事项

1. **保守优先**：对于拿不准是否为错别字的词，**不修改**，在汇总中用 `⚠️ 存疑词汇` 单独列出供用户确认
2. **不润色**：只改错字，不改写句子，不优化表达，不替换同义词
3. **原格式优先**：输出的完整文本必须与输入格式保持一致（LaTeX 仍是 LaTeX，Markdown 仍是 Markdown）
4. **术语宽松认定**：遇到不确定是否为术语的技术词汇，倾向于标记为术语而非修改
